# US-MOD-03: одобрение товара модератором

## Повторная сдача

Этот PR является отдельным артефактом для повторной сдачи US-MOD-03 после замечаний по исходящему событию в B2B.

Проверяемые требования:

- approve выполняется через `POST /api/v1/products/{product_id}/approve`;
- карточка должна быть в `IN_REVIEW`;
- карточка должна быть закреплена за текущим модератором;
- товар без SKU не одобряется;
- успешное решение переводит карточку в `MODERATED`;
- в B2B отправляется событие по контракту `ModerationEventRequest`.

## Исправленный outbound contract

Для B2B используется путь:

```text
POST /api/v1/moderation/events
```

Payload содержит обязательные поля OpenAPI:

```json
{
  "idempotency_key": "...",
  "product_id": "...",
  "event_type": "MODERATED",
  "occurred_at": "..."
}
```

В payload больше нет старого поля `status`.

## ADR

Рассматривал три способа доставки события MODERATED в B2B: синхронный POST в обработчике approve, outbox-pattern с отдельным воркером и event-bus. Для первой итерации выбран синхронный POST плюс запись в локальный outbox: модератор сразу видит ошибку доставки, а событие остаётся наблюдаемым в БД. Полный outbox с retry надёжнее, но требует фонового процесса. Event-bus масштабируемее, но избыточен для текущего учебного сервиса.
